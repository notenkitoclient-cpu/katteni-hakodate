# 【管理部】システム全体のステータスを確認する

カッテニハコダテの全パイプラインの現状を横断的にレポートする。

## 確認内容

以下をすべて調べて、部署別にまとめて報告してください：

### 収集部

- SNSキュー未処理画像数 (`src/content/sns-queue/`)
- 直近7日間に収集されたニュース数 (`src/content/news/`)
- スプレッドシート処理済みID数 (`src/data/sheets-processed-ids.json`)

### 編集部

- 下書き件数 (`src/content/drafts/`)
- 公開済み記事件数 (`src/content/articles/`)
- 記事キューの残件数 (`src/content/article-queue.txt`)

### 配信部

- 直近公開記事（最新3件のタイトルと日付）

### 管理部

- Git の未コミット変更の有無
- 最後のコミット日時とメッセージ

## 問題の検出

以下の状態を検出した場合は警告を出す：
- 下書きが7件以上溜まっている（公開が詰まっている）
- ニュースが3日以上収集されていない（収集部が止まっている）
- SNSキューに未処理画像が残っている（処理漏れ）

最後に「次にすべきアクション」を1〜3件提示する。
